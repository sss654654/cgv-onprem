# frontend — 예매 SPA + nginx 게이트웨이

브라우저가 접근하는 진입점이다. 바닐라 JS 단일 페이지 앱(SPA)과, 정적 파일을 서빙하면서 API를 뒤 서비스로 넘기는 nginx 리버스 프록시로 구성된다. 빌드 도구·프레임워크가 없다(HTML·CSS·JS 3파일).

- 화면: 영화목록 → 대기 → 회차선택 → 좌석 → 결제 → 완료
- 게이트웨이: `/api/admission/*` → queue / 나머지 `/api/*` → booking
- 상태 전달: 서버 push 없음 — 클라이언트가 `GET /position`을 폴링

---

## 현재 상태

로컬 `docker-compose`에서 nginx 게이트웨이 겸 정적 서버로 동작한다. k3s에서는 라우팅 역할을 Traefik Ingress가 대신하며, nginx는 정적 파일만 서빙하는 파드로 남는다(경로 규칙은 그대로 이관). 폰트는 외부 CDN(jsDelivr의 Pretendard)을 참조하므로 오프라인·폐쇄망에서는 시스템 폰트로 대체된다.

---

## 시스템에서의 위치

```
   브라우저  (http://localhost:8088)
     │
     ▼
 ┌──────────────────────────────┐
 │  frontend (nginx, 이 서비스) │   정적: index.html · app.js · style.css
 │  = 정적 서버 + 리버스 프록시  │
 └───────┬──────────────┬───────┘
         │ /api/admission/*     │ 나머지 /api/*
         ▼                      ▼
      queue (8090)          booking (8091)
```

브라우저는 이 한 주소만 본다. 경로 접두로 뒤 서비스가 갈리며, 더 구체적인 `/api/admission/`이 `/api/`보다 우선한다. k3s에서는 이 규칙이 Ingress 매니페스트로 옮겨간다.

---

## 화면 흐름 (상태 머신)

```
        [로그인]  requestId 발급(localStorage)
            │
            ▼
   ┌──── movies ────┐  영화 클릭 → enter
   │        │                       │
   │        │  202 WAITING          │ 200 ADMITTED
   │        ▼                       │
   │     waiting  ──ADMITTED발견──▶ entering ──▶ screenings ──▶ seats ──▶ pay ──▶ done
   │   (순번 폴링)      (전환 유예)   (회차 선택)   (좌석 선택)  (결제)   (완료)
   │        │                                        │
   │        └── EXPIRED(타임아웃·이탈) ──────────────┘
   │                                                 │
   └──────────────────── 홈/뒤로 ────────────────────┘
```

- **waiting → entering**: 폴링 응답이 `ADMITTED`로 바뀌면 곧장 좌석으로 가지 않고 "입장 중…" 화면을 잠깐 거친다. queue의 승격과 booking의 admitted 채움 사이 시간차 때문에 좌석 조회가 403이 날 수 있어서, 그 사이 회차 조회를 재시도(최대 12회·0.3초 간격)해 흡수한다.
- **입장 후에도 느린 폴링 유지**: active가 된 뒤에도 5초 간격으로 `/position`을 계속 조회한다. 서버가 세션 타임아웃으로 자리를 회수하면(`EXPIRED`) 그것을 감지해 홈으로 돌린다.

## 클라이언트가 담당하는 것

서버가 stateless·폴링이라, 다음은 전부 클라이언트 몫이다.

| 관심사 | 구현 |
|---|---|
| 신원 | `requestId` = 로그인 시 발급하는 UUID, localStorage 지속. 모든 요청에 실린다 |
| 순번 조회 | `GET /position` 폴링. 순번이 뒤일수록 주기를 늘리고(1/2/5초), jitter를 더해 동시 폴링 herd를 분산 |
| 403 경쟁 | 입장 직후 좌석/회차 403을 치명 오류로 안 보고 짧게 재시도(비동기 시간차 흡수) |
| 카운트다운 | active 1분 타이머는 표시용. 실제 기준은 서버 세션 타임아웃이라, 로컬 타이머는 서버 감지 지연보다 넉넉한 마진 뒤에만 퇴장시킨다 |
| 새로고침 복구 | 진행 상태를 `cgv_flow`에 저장 → F5 후 같은 requestId로 enter 재호출(서버: active=유지 / waiting=꼬리로 밀림) |
| 네비게이션 정합 | 뒤로가기·홈은 화면만 바꾸지 않고 서버 상태를 맞춘다 — 좌석락 해제(`/release`)와 자리 반납(`/leave`)을 짝으로 호출. 안 하면 TTL까지 점유가 남는다 |

응답 상태코드별 화면 처리: 202→대기, 200→입장, 409→그 좌석만 회색+알림(전체 새로고침 없음), 403→미승인(재시도 또는 퇴장), 503 `RETRY_LATER`→지연 안내(화면 유지).

## 코드 구조

```
frontend/
├── index.html     화면 6종의 마크업(section) + 시뮬레이터 패널
├── app.js          상태 머신·API 호출·폴링·네비게이션 전부
├── style.css       스타일(다크 톤)
├── nginx.conf      라우팅 3분기 + SPA fallback + no-cache
└── Dockerfile      nginx-unprivileged(비루트 uid 101, 8080)
```

`app.js` 하나에 앱 상태(`S`), 화면 전환(`show`), API 래퍼(`api` — 네트워크 예외까지 잡아 폴링이 멈추지 않게 함), 폴링 루프, 좌석 그리드 렌더가 모두 있다. 별도 상태관리 라이브러리·번들러가 없다.

---

## 시뮬레이터 (데모 도구)

영화목록 화면에만 나오는 개발/시연용 패널이다. 가짜 `requestId`로 enter를 여러 번 호출해 대기열을 채운다. "봇 자동예매"를 켜면 가짜 사용자들이 3초 틱마다 `/position`을 폴링하고(폴링이 곧 생존 신호라 대기열이 유지된다), 입장되면 스스로 좌석을 잡고 예매하고 나간다 — 자리 순환이 화면에 표시된다. 봇 폴링이 없으면 가짜 대기자가 30초 만에 전부 회수돼 대기열이 사라진다.

## 게이트웨이 규칙 (nginx.conf)

```
location /api/admission/   → proxy_pass http://queue:8090     (입장 통제)
location /api/             → proxy_pass http://booking:8091   (예매)
location /                 → try_files → index.html           (SPA fallback)
                             + Cache-Control: no-cache
```

- 폴링 전환 후 SSE 스트림용 설정(버퍼 off·긴 타임아웃)은 제거했다. 짧은 요청·응답이라 불필요하다.
- 파일명에 해시가 없어 캐시버스팅 수단이 없으므로 `no-cache`로 재배포 시 구버전 노출을 막는다(조건부 요청이라 304로 처리된다).
- k3s에서는 Ingress가 `/api`를 먼저 가로채므로 이 proxy 규칙은 타지 않는다(그대로 둬도 무해).

---

## 실행

전체 스택은 루트에서 `docker compose up --build -d`로 띄운다. 브라우저 진입점은 http://localhost:8088. 빌드 단계가 없어 파일을 이미지에 복사만 하면 된다.

## 알려진 한계 / 다음

- **디자인 미니멀**: 동작 우선의 바닐라 구현. 추후 재설계 가능.
- **폰트 CDN 의존**: 외부 CDN 참조라 폐쇄망에서는 폴백된다. 자산 인라인/self-host는 미적용.
- **좌석 폴링**: 좌석도는 5초 간격 새로고침으로 갱신한다(정확성은 서버의 409가 보장하므로 지연은 무해).
